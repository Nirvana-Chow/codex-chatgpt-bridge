# 给 ChatGPT 的训练规划公开上下文

> 这是给 ChatGPT 讨论训练规划用的公开安全上下文。它可以进入 GitHub bridge。不要在这里放训记 API Key、账号、原始训练数据、完整身体历史、工作、财务、人物朋友或私密日记。

## 想让 ChatGPT 帮忙看的问题

- 减脂优先时，训练量、动作选择和有氧安排是否合理。
- 飞盘可能当天发生时，是否应该避开重腿和高强度腿部间歇。
- 上肢健美化、核心抗旋、心肺、柔韧和飞盘专项如何排优先级。
- 训记写入动作是否应该用更精确或更适合统计的标准动作名。
- 如何把长期目标拆成每天可执行的计划，而不是一次性写死。

## 公开范围

可以公开：

- 训练目标、训练原则、当天计划、动作名、组数、训练目的。
- 为训练讨论必要的粗身体基线：身高约 170 cm，体重约 83-84 kg，体脂估计约 26%，腰围待测。
- 训练场景：健身房、飞盘、跑跳、雪橇、瑜伽垫、弹力绳、泡沫轴、筋膜球、筋膜枪。

不公开：

- 训记训练、饮食、身体数据 API Key。
- 训记原始历史数据、完整体重体脂导出、饮食明细。
- 工作、财务、朋友/人物关系、账号密码、私密日记和本地文件路径。

## 训练目标

优先级从高到低：

1. 减脂：到 2026-09-06 同学婚礼前明显改善体脂和形体状态。
2. 保持力量：减脂过程中保住卧推、划船、下拉/引体、下肢基础力量。
3. 提升心肺：服务飞盘反复跑动、恢复和减脂。
4. 提升飞盘能力：启动、急停、变向、落地、核心抗旋、肩胛控制和投掷稳定。
5. 提升柔韧与活动度：降低飞盘受伤风险，长期向劈叉、下腰、倒立推进。
6. 保留健美化体型：胸、背、肩、臀、手臂有训练痕迹，整体均衡。

## 核心训练原则

- 以用户自己作为训练主体：没有飞盘局时，也要能靠健身房和自练提高底层能力。
- 每天按状态排当天计划，不把一周模板一次性写死。
- 飞盘待定或可能当天打飞盘时，避免重腿、高容量腿、雪橇 HIIT 和冲刺间歇。
- 腿部要强、稳、弹，但不追求为了围度把腿练废；飞盘表现优先于短期腿部泵感。
- 核心抗旋属于功能性训练和飞盘专项基础，用于传盘、急停、变向和对抗中的躯干稳定。
- 泡沫轴和筋膜枪用于进入训练状态和放松，不替代力量训练、活动度训练和睡眠恢复。

## 训练模块

| 模块 | 目的 | 常见内容 |
| --- | --- | --- |
| 力量训练 | 保住基础力量和肌肉量 | 卧推、深蹲、罗马尼亚硬拉、引体/下拉、划船 |
| 形体健美 | 让身体有训练痕迹 | 上胸、背宽背厚、侧束、后束、臀、手臂 |
| 飞盘专项 | 让力量转移到场上动作 | 启动、急停、横移、变向、起跳、落地 |
| 心肺训练 | 减脂和反复跑动能力 | Zone2、间歇、雪橇、单车、椭圆机 |
| 功能性训练 | 稳定和传力 | 核心抗旋、单腿稳定、肩胛控制、髋踝活动度 |
| 柔韧技能 | 活动度和长期技能 | 劈叉线、桥线、倒立线、肩胸和髋踝打开 |
| 恢复 | 让训练可持续 | 睡眠、低强度活动、泡沫轴、拉伸、降量 |

## 训记动作名规则

训记写回必须使用标准中文动作名，来源：

https://github.com/Foveluy/Xunji-movements

规则：

- 写回训练时只传中文 `name`，不传内部 key。
- 不写斜杠动作名，例如不要写 `杠铃卧推 / 哑铃卧推`。
- 如果现场改动作，训练后按实际动作改成另一个标准动作名。
- 单条训练最多 15 个动作，每个动作最多 20 组；小热身和手动拉伸可以放备注，不一定都占动作位。
- 有氧动作用训记表里的有氧标准名，例如 `椭圆机(计时)` 或 `跑步_有氧训练`。

## 长期一周框架草案

- D1：上肢力量与体型。
- D2：下肢专项与飞盘能力。
- D3：心肺、功能性与柔韧。
- D4：全身体型与综合能力。
- 有飞盘局时，飞盘替代 D2/D3 中高强度专项，不额外硬加。
- 忙时做 3 练：上肢力量形体、下肢专项、全身心肺功能性。

## 训练与数据闭环

- 训记：保存动作、组、重量、次数、RPE、有氧、饮食和身体指标。
- Obsidian：保存目标、计划、判断、复盘、报告和长期上下文。
- Codex：每天根据状态生成当天计划；写回训记前必须先展示摘要并等待用户确认。
- ChatGPT：用于讨论计划是否合理、提出改进意见和评审训练逻辑。

## 给 ChatGPT 的评审要求

- 哪些动作或组数对减脂、形体、飞盘兼容性最有价值。
- 今天飞盘待定时，上肢训练是否比下肢训练更合理。
- 是否应该减少热身/拉伸在训记里的动作数量，改成备注。
- 如果要兼顾“减脂、飞盘、柔韧、健美化”，一周微周期如何更稳。
- 哪些动作名可能不适合训记统计，应该换成更标准的动作。
